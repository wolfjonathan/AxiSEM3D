# Example 08: Global seismoacoustics — the Mars atmosphere

This example demonstrates seismoacoustic wave propagation coupled between a
planetary atmosphere and the solid interior, using Mars as the test body. The
model is linear, static-background and gravity-free: it cannot represent winds
or atmospheric gravity waves, but it does capture the linearised pressure-wave
coupling of an explosion in the atmosphere into the ground (or the reverse)
across a chosen frequency band. The uppermost layer of the background model is
an acoustic atmosphere (defined by `vs = 0`, `vpv = vph`), and an absorbing
(Kosloff–Kosloff sponge plus Clayton–Enquist) boundary is applied at the top of
the atmosphere to suppress spurious reflections. The same recipe transfers to
Earth, Titan or other bodies by editing the background model. A `FLUID_PRESSURE`
source injects an overpressure pulse, and ten surface receivers along a meridian
record the coupled ground motion.

Because the model is **1D axisymmetric** and the source and all receivers lie on
a single meridian, the **transverse (T) component is identically zero by
symmetry**. The flat T panel in the seismogram figure is *physics, not a bug*.

## Contents

- `axisem3d_mars_atm/input/inparam.model.yaml` — 1D model, geodesy, and the
  top-boundary absorbing condition (`boundaries: [TOP]`, Kosloff–Kosloff sponge
  with `relative_spans: [0.01]`); references the mesh `tayak_60km_20s.e`.
- `axisem3d_mars_atm/input/inparam.source.yaml` — time axis (record length
  1800 s, Courant 0.5) and a single `FLUID_PRESSURE` source on the axis at
  radius 3 394 000 m with a Gaussian source-time function.
- `axisem3d_mars_atm/input/inparam.nr.yaml` — azimuthal resolution (Nr) settings.
- `axisem3d_mars_atm/input/inparam.output.yaml` — station group `MARS` reading
  `mars.txt`, RTZ displacement (`channels: [U]`), `ASCII_STATION` format.
- `axisem3d_mars_atm/input/inparam.advanced.yaml` — advanced/MPI settings.
- `axisem3d_mars_atm/input/mars.txt` — 10 surface receivers along a meridian
  (network `A`, latitudes 0°–9°, longitude 0°, 1 m below the solid surface).
- `axisem3d_mars_atm/tayak_60km_atm.bm` — background model: TAYAK Mars interior
  plus a 60 km acoustic atmosphere above the 3389.5 km surface radius (the
  uppermost 30 km is constant-density "padding" so the sponge layer stays in
  the unphysical part of the atmosphere).
- `axisem3d_mars_atm/tayak_60km_atm_meshing.bm` — the mesher-ready model written
  by `prepare_positive_atmosphere_bm.py` (atmosphere branch densified onto a
  positivity-preserving radius grid); regenerated by `gen_mesh.sh`.
- `axisem3d_mars_atm/prepare_positive_atmosphere_bm.py` — rewrites the
  atmosphere branch of the `.bm` onto a denser grid, interpolating density in
  log-space so it stays strictly positive.
- `axisem3d_mars_atm/validate_exodus_fluid_rho.py` — checks that the generated
  Exodus mesh has strictly positive fluid `RHO_*` corner fields before running.
- `axisem3d_mars_atm/gen_mesh.sh` — end-to-end mesh pipeline (prepare → mesh →
  validate).
- `axisem3d_mars_atm/README.txt` — the original author's detailed notes on the
  model, the absorbing boundary, and the fluid source.
- `make_figures.py` — post-processing (see *Reproduce* below).
- `figures/` — pre-rendered reference figures (record section, three-component
  seismograms, source-receiver map, and the surface-wavefield overview).

## Requirements

You need a built **AxiSEM3D** solver. Either install the `axisem3d` binary on
your `PATH`, or copy it into this example as `./axisem3d`; the commands below
assume it is on the `PATH`. Mesh generation needs `SalvusMeshLite`
(`salvus_mesh_lite`); the figures need Python with `numpy` and `matplotlib`
(plus `cartopy` for the globe maps — the script falls back to a flat lat/lon
view if `cartopy` is absent).

## Reproduce

### 1. Mesh

The shipped configuration uses the **20 s** mesh `tayak_60km_20s.e`. That mesh
exceeds the 4 MB size cap for this repository and is therefore **not** included;
regenerate it with the example-local pipeline, which densifies the atmosphere
branch, meshes it, and validates fluid density:

```bash
cd axisem3d_mars_atm
./gen_mesh.sh 20            # writes input/tayak_60km_20s.e (matches inparam.model.yaml)
```

`gen_mesh.sh <period> [output_file]` runs three steps:

1. `prepare_positive_atmosphere_bm.py` rewrites `tayak_60km_atm.bm` onto a dense,
   positivity-preserving radius grid → `tayak_60km_atm_meshing.bm`.
2. SalvusMeshLite meshes the prepared model at the requested period:
   ```bash
   python -m salvus_mesh_lite.interface AxiSEM \
       --basic.model tayak_60km_atm_meshing.bm \
       --basic.period 20 \
       --output_filename input/tayak_60km_20s.e
   ```
3. `validate_exodus_fluid_rho.py` confirms all fluid `RHO_*` corner values are
   strictly positive.

To refine, pass a smaller period (e.g. `./gen_mesh.sh 10` for a 10 s mesh) and
update `exodus_mesh:` in `inparam.model.yaml` to the new filename. Higher-period
(coarser) meshes are smaller; lower-period (finer) meshes are larger.

### 2. Large data

None beyond the mesh in step 1. The `.bm` background models ship with the
example; the only large artefact is the Exodus mesh, which you regenerate
locally (above) rather than download.

### 3. Run

Run the solver from the simulation directory, pointing AxiSEM3D at the `input/`
directory (which now contains the regenerated mesh). The 20 s mesh is coarse, so
a handful of MPI ranks is plenty:

```bash
cd axisem3d_mars_atm
mpirun -np 4 axisem3d --input input --output output
```

Seismograms are written to `output/stations/MARS/` (10 ASCII files `A.0`…`A.9`
plus `data_time.ascii`). Increase `-np` to the core count you have available.

On a cluster, wrap the `mpirun` command in your scheduler's job script and load
your compiler, MPI, netCDF and HDF5 modules.

### 4. Figures

After the run, generate the figures with:

```bash
python make_figures.py
```

`make_figures.py` reads the station output this example produces
(`axisem3d_mars_atm/output/stations/MARS/`, set by the `INPUT_DIR` variable at
the top of the script) together with the station-info file
`axisem3d_mars_atm/input/mars.txt` (for epicentral distances). It locates both
relative to itself and imports the shared plotting helper `pub_style.py` from the
top-level `examples/` directory (one level above this example). It is headless
(matplotlib Agg backend) and writes four figures into `figures/` as **PDF + PNG**:

| Figure file (in `figures/`) | What it shows |
|---|---|
| `ex08_mars_record_section` | Vertical-component (Z) record section; traces normalised and offset by epicentral distance (0°–9°), one per station. |
| `ex08_mars_seismograms` | Three-component (R/T/Z) seismograms at a representative low-distance station; R/T/Z share one y-scale so the zero transverse trace reads as flat. Δ is printed in the title. |
| `ex08_source_receiver_map` | Source-receiver geometry on the Mars globe (no Earth coastlines). Requires `cartopy`; skipped with a note if it is missing. |
| `ex08_mars_overview` | Orthographic surface-wavefield snapshots over the resolved near-axis cap, beside the record section. Falls back to a flat lat/lon view without `cartopy`. |

Seismograms are plotted in **metres**; channel colours are colorblind-safe
(Okabe-Ito, from `pub_style.py`): Radial = vermillion, Transverse = bluish-green,
Vertical = blue. The pre-rendered versions of all four figures are included in
`figures/` for reference.
